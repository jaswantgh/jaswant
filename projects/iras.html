Box Terminal Maze Part 1

  • Tech Stack: C89
  • Github URL: Project Link

> In my second year, I received an assignment in Unix and C Programming to develop a box terminal maze game. The game, termed "box", mandated six parameters to set the map size, player's location, and the goal's position. Beyond handling errors for improper input and concluding the program with apt messages, I was also tasked with the following:

> Firstly, the assignment required mastery of command-line arguments, which dictated the game's dynamics. A video tutorial aided in comprehensively understanding how to implement this.

> Secondly, I had to incorporate a random number generator into the game. This introduced unpredictability into the gameplay, enhancing the challenge.

> A vital aspect of the assignment was the introduction of a 'PULL' function, facilitated through conditional compilation. This unique feature allowed the player to pull the box within the maze.

> Furthermore, the assignment demanded specific knowledge on clearing the screen and implementing the scanf function for immediate character input, adding to the game's user-friendly interface.

> Lastly, to maintain high code quality, I made use of Valgrind to debug and correct any potential memory errors or leaks.

> In the end, the complexity and rigor of this assignment led me to achieve a perfect score of 100%.
