Box Terminal Maze Part 2

  • Tech Stack: C89

> Following the successful completion of my first Unix and C Programming assignment, I moved on to the second task which built upon the foundations of the first one.

> In this assignment, I was tasked with the addition of several new features to the box terminal maze game. The added features included file I/O operations, a color trail, and an undo feature.

> Firstly, I implemented file I/O operations which facilitated saving and loading game states. This provided the capability for players to pick up where they left off, enhancing the user experience.

> Next, I added a color trail to the game. This visual aspect allowed players to see the path they had traveled, serving as a valuable navigation aid within the maze.

> The third new feature was the introduction of an undo function. This gave players the opportunity to correct their moves, further improving gameplay interaction.

> In addition to coding these features, I continued to apply rigorous debugging and memory checks using Valgrind, ensuring the integrity of the program.

> This more complex assignment, filled with layers of enhancements to the basic game, was a great learning experience. I was rewarded for my meticulous efforts with a high score of 98%, marking yet another significant milestone in my Unix and C Programming journey.
